📖 Энциклопедия: JSON, YAML и Архитектура

                
                
1. Введение: JSON и YAML

                
JSON (JavaScript Object Notation) — строгий стандарт передачи данных для машин. Он быстрый и однозначный.
YAML (YAML Ain't Markup Language) — надстройка над JSON, созданная для удобного чтения людьми. Часто используется для конфигурации серверов.

                
                
2. Структура JSON: Объекты и Массивы

                
Данные хранятся в двух видах:
Объект { }: Набор пар «Ключ: Значение».
Массив [ ]: Упорядоченный список (как список покупок).

                
{"name": "Иван"}
[1, 2, 3]

                
                
3. Типы данных JSON

                
JSON поддерживает только 6 типов данных:

                
"Строка" (текст)
42 (число)
3.14 (дробь)
true/false (логическое)
null (пустота)


                
4. Строгие правила и Ошибки

                
JSON не прощает ошибок. Ключи всегда в "двойных кавычках". Одинарные кавычки (' ') вызовут краш. Висячие запятые перед } или ] строго запрещены.

                
{"name": "Иван"} // Правильно
{name: "Иван"}   // ОШИБКА
[1, 2, 3,]       // ОШИБКА

                
                
5. Экранирование спецсимволов

                
Если в тексте нужна кавычка, ставим перед ней слеш. Перенос строки — \n, табуляция — \t.

                
\" — двойная кавычка
\\ — слеш
\n — перенос текста

                
                
6. Форматирование (Minify vs Pretty)

                
Для компьютера пробелы и переносы не нужны. Сжатый код работает так же, но экономит память (Minified).

                
{"a":1,"b":2} // Сжатый (Minified)

{\n  "a": 1,\n  "b": 2\n} // Красивый (Pretty)

                
                
7. YAML: Синтаксис и Отступы

                
YAML отказывается от скобок и кавычек. Структура строится за счет строгих пробелов (Tab запрещен!). Массивы обозначаются дефисом.

                
# Комментарий
server: Proxy
dns:
  - 8.8.8.8


                
8. YAML: Многострочный текст

                
Символ | сохраняет все переносы строк точно так, как вы их написали. Символ > склеивает все переносы в одну длинную строку.

                
message: |
  Первая строка.
  Вторая строка.

                
                
9. Что такое Base64 и HAPP?

                
Base64 — это алгоритм перевода любых данных в безопасный текст (только латиница и цифры).
HAPP-ссылка (happ://routing/onadd/...) — это стандарт для быстрой передачи настроек в VPN-клиенты (v2ray, Sing-box). Клиент читает ссылку, декодирует Base64 и получает готовый JSON.

                
                
10. Шаринг и Iframe

                
Кнопка "Поделиться" кодирует твой код в Base64 прямо в URL-ссылку браузера (серверы не нужны!). Код Iframe позволяет вставить этот редактор прямо в HTML твоего блога.
